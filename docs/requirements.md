**Техническое задание:**

1. **Основной функционал:**
   - Регистрация и аутентификация пользователей
     - внешняя идентификация по email
     - внутренняя - id в СУБД
   - Управление проекатми
     - Проект "Public"
       - используется для незарегистрированных пользователей
       - максимальный срок жизни ссылок в проекте Public - 5 дней
       - операции в проекте ограничены следующими:
         - создание новых ссылок
         - просмотр ссылок из проекта Public
         - просмотр ссылок, отмеченных как "public" в других проектах  
     - При регистрации пользователя создается "личный" проект
     - В проект может входить несколько пользователей
     - Один пользователь может входить в несколько проектов
     - У проекта есть один или болеее администраторов из числа пользователей проекта
       - по-умолчанию тот, кто создал проект
       - только администратор может добавлять в проект пользователя, указаы email
       - только администратор может назначать роль администратора проекта
     - Изменить проект может только пользователь проекта с ролью администратора (удалить проект, удалить одну или несколько ссылок, изменить связь короткой ссылки с оригинальным URL)
     - для проекта задается время жизни ссылок по-умолчанию (дни)
   - Управление ссылками (CRUD операции, обязательные для реализации)
     - `POST /links/shorten` - генерация короткой ссылки и связывание с переданным длинным URL
       - опциональный параметр `expires_at` в формате даты с точностью до минуты - после указанного времени короткая ссылка автоматически удаляется.
       - опциональный параметр `custom_alias` - создание ссылки с заданным алиасом, если он свободен.
       - опциональный параметр `project_id` 
         - включение ссылки в указанный проект, если пользвоатель является членом проекта
         - по-умолчанию используется личный проект пользователя
     - `GET /links/{short_code}` - перенаправление на оригинальный URL
     - `PUT /links/{short_code}` - обновление свяанного URL
     - `DELETE /links/{short_code}` - удаление связи
   - Статистика по короткой ссылке
     - `GET /links/{short_code}/stats` - возвращает оригинальный URL, дату создания, количество переходов, дату последнего использвоания
   - Поиск ссылки по URL
     - `GET /links/search?original_url={url}`

2. **Дополнительные функции:**
   - Автоматическая очистка истекших ссылок
   - Экспорт статистики
   - Bulk операции с ссылками (реализуется через UI)

3. **Технические требования:**
   - FastAPI как основной фреймворк
   - PostgreSQL для основного хранилища
   - Redis для кэширования
   - Docker для контейнеризации
     - Отдельные контейнеры для:
       - FastAPI приложения
       - PostgreSQL
       - Redis
       - Streamlit - интерфейс
     - Docker Compose для оркестрации
   - Развертывание на Yandex Cloud
   - Test Driven Development
     - pytest как фреймворк тестирования
     - 100 покрытие тестами

**Предлагаемая архитектура:**

1. **Основные компоненты:**

   a) **API Layer:**
   - REST API endpoints для всех операций с ссылками, пользователями, проектами
   - Middleware для аутентификации
   - Валидация входных данных

   b) **Service Layer:**
   - Бизнес-логика для работы с ссылками, пользователями, проектами
   - Генерация коротких кодов
   - Управление кэшем

   c) **Data Layer:**
   - PostgreSQL для хранения:
     - Ссылок (оригинальный URL, короткий код, статистика)
     - Пользователей
     - Проектов (для группировки ссылок)
   - Redis для кэширования:
     - Популярных ссылок
     - Статистики переходов
     - Временных данных

    d) **User Interface:**
    - Streamlit для интерфейсов:
      - пользователя
      - администратора всей системы

2. **Ключевые особенности реализации:**

   a) **Кэширование:**
   - Кэширование популярных ссылок в Redis
   - Кэширование статистики переходов
   - Инвалидация кэша при обновлении/удалении

   b) **Безопасность:**
   - JWT аутентификация
   - Rate limiting
   - Валидация URL

   c) **Мониторинг:**
   - Логирование запросов
   - Метрики использования
   - Отслеживание ошибок
